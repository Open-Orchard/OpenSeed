App Data

Application data on OpenSeed is handled the same way we handle all data inputs. JSON formated data in, JSON formated data out. There are two types of App data storage Private (As in encrypted) and Public (which is stored in plain text). We seperate these two for various reasons but the easiest example as to why we would have both encrypted and un-encrypted databases is to allow for easier cross application data sharing and faster searches. It could also be used for applications that wish to have a blog like service or accountless access to some data.

info

All application data is sent to the server already encrypted, using this encryption key we determine if an application has the right to store data.

#Data Management Functions

Beyond private and public stores the developer needs to set whether this data is going to overwrite existing data (update) or have a seperate database entry.

tip

OpenSeed doesn't care about the data that is stored but it is always wise to maintain the json-in json-out standard we impose on other functions to limit the amount of work you'll have using these features.

#Storing Private Data

This data will be stored using the private ID assigned to the application.

warning

Sense this data is encrypted the validity of the data can not be checked by the server. The Developer will need to be sure to handle the decryption and formating on their end.

Call

msg = {
"act":"set_private_data"
"devid": ""
"appid": ""
"data":""
"update":boolean

Returns

  • Success

    {server:{"appdata":"updated/added"}}

#Storing Public Data

Public data can be search by any application using OpenSeed. Private information should never be stored publically. Likewise even encrypted data stored in the public data can be accessed by other applications.

tip

Unlike Private data, we will check to make sure the data is formated in a standard JSON compliant manner. This ensures that search features work.

Call

msg = {
"act":"set_public_data"
"devid": ""
"appid": ""
"tag":""
"data":""
"update":boolean

Returns

  • Success

    {server:{"appdata":"updated/added"}}

  • JSON Error

    {"server":{"appdata":"not json"}}

Retrieving Data

Privately stored data is only retrievable by the application that stored it. Publicly retriviable data is searchable by all applications on OpenSeed. Keep this in mind as you design your data persitance strategies.

info

Due to the nature of the command we can retreive private and public data using the same request.

Call

msg = {
"act":"get_app_data"
"devid": ""
"appid": ""
"tag":""
"mode":"private/public"

Returns

  • Success

{"appdata":{"tag":"'+tag+'","data":'+data+'}}

  • No Tag

{"appdata":{"tag":"error","data":"error"}}
