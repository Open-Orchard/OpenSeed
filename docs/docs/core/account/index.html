Accounts

Current existing functions dealing with OpenSeed Accounts. Please read the About section to understand how to use the below in a fully formated command. Not all these functions are finalized and there will be changes while we work toward our first release. However, these should represent most of the major features.

#General Account Functions

  • Account Check:

    Checks user accounts returns true if correct.

    msg=
    {
    "act":"account_check",
    "devid":"",
    "appid":"",
    "accout":"",
    "passphrase":""
    }

  • Create a User:

    Creates Account based on the included criteria

    msg=
    {
    "act"="create_account"
    "devid"="",
    "appid"="",
    "account"="",
    "passphrase"="",
    "email"=""
    }

  • Create a Creator Account:

    Creates Developer account

    msg=
    {
    "act"="create_creator_account"
    "devid"="",
    "appid"="",
    "devName"="",
    "contactName"="",
    "contactEmail"="",
    "openseed"=""
    }

  • Create App or Game:

    Creates Application and associates it with the developer.

    msg=
    {
    "act"="create_app"
    "devid"="",
    "appid"="",
    "appName"=""
    }

  • Create Profile:

    Creates the long form profile for users,developers, and applications. Called after the ids have been created.

    msg=
    {
    "act":"set_profile",
    "devid":"",
    "appid":"",
    "token":"",
    "data1":"",
    "data2":"",
    "data3":"",
    "data4":"",
    "data5":"",
    "type":""
    }

  • Add to History :

    Adds to the history record for the user

    msg=
    {
    "act":"update_history",
    "token":"",
    "type":"",
    "appPub":"",
    "data":""
    }

  • Get History :

    Returns the history for a specific account. Optionally you may define a specific application and a total count.

    msg=
    {
    "act":"get_history",
    "devid":"",
    "appid":"",
    "account":"",
    "apprange":"",
    "count":""
    }

#Hive Account Functions

  • Hive OpenSeed Interconnect :

    This creates an openseed user based on the hive profile retrieved on verification of there hive account, gives openseed posting rights, and gives the user the option to have their keys stored. This is the basis for logins and other hive/openseed functions.

    msg=
    {
    "act":"hive_openseed_interconnect",
    "devid":"",
    "appid":"",
    "account":"",
    "postkey":"",
    "store":true/false
    }

  • Hive Delegation :

    If the user has allowed OpenSeed to store their keys we can quickly add or remove posting rights through the hive_delegation function. If they don't have their keys saved by OpenSeed we will need to ask them for their posting key to finish the process. We derive the application we are allowing/disallowing via appid.

    msg=
    {
    "act":"hive_delegation",
    "devid":"",
    "appid":"",
    "account":"",
    "postkey":"" -or- "passphrase":"",
    "option":"allow"/"disallow"
    }
note

This function will attempt to streamline login prompts similar to steemconnect or hivesigner, but steps will need to be taken to create openseed accounts using hive_openseed_interconnect if the developer is hoping to get the login to a point where it behaves like other login managers. (i.e. facebook,google,etc.)

  • Flush Keys (Hive) :

    Not needed for most operations as the deletion of keys is a part of every request that requires one (unless stored is set true). However, when OpenSeed creates an account for a user the keys are stored for fast access. This function can be used to remove the keys from our server.

    msg=
    {
    "act":"hive_flush_keys",
    "devid":"",
    "appid":"",
    "hiveaccount":"",
    }

  • Send Payment (Hive) :

    Returns the history for a specific account. Optionally you may define a specific application and a total count.

    msg=
    {
    "act":"hive_send_payment",
    "devid":"",
    "appid":"",
    "account":"",
    "apprange":"",
    "count":""
    }

  • Get Account (Hive):

    retrieves the users profile from hive node

    msg=
    {
    "act":"get_hive_account",
    "devid":"",
    "appid":"",
    "account":"",
    }

  • Get Full Account (Hive):

    retrieves all user info from hive node

    msg=
    {
    "act":"get_hive_account",
    "devid":"",
    "appid":"",
    "account":"",
    }
