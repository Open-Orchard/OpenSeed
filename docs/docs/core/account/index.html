Accounts

Current existing functions dealing with OpenSeed Accounts. Please read the About section to understand how to use the below in a fully formated command. Not all these functions are finalized and there will be changes while we work toward our first release. However, these should represent most of the major features.

#General Account Functions

#Account Check:

Checks user accounts returns true if correct.

Call:

msg=
{
"act":"account_check",
"devid":"",
"appid":"",
"accout":"",
"passphrase":""
}

Returns:

  • Success:

    {"account":{"token":"user token","username":"username"}}

  • Invalid username/password

    {"account":{"token":"denied"}}

  • No user found

    {"account":{"token":"none"}}

#Create a User:

Creates Account based on the included criteria. Along with creating the tokens it creates a mostly empty profile that is returned on creation to help developers with the next steps.

note

The profile creation is mandatory but an option might be added to create_account to supress the profile information.

Call:

msg=
{
"act"="create_account"
"devid"="",
"appid"="",
"account"="",
"passphrase"="",
"email"=""
}

Returns:

  • Success:

    {"account":{"token":"'+uid+'","username":"'+username+'","profile":'+pfile+'}}

  • User exists:

    {"account":{"username":"exists"}}

#Create a Creator Account:

Creates Developer / Creator account. In OpenSeed we seperate normal users and creator accounts to allow for multiple creator instances.

info

We call all producers "creators" as OpenSeed wants to avoid multiple user accounts whenever possible. (e.g. jondoe has a creator account name "Doe Ray ME" where he preforms one man musicals. He also has a creator account called "Software made by Doe" where he publishes his software. Both are connected to the account jondoe which is inturn connected to a wallet account that allows all his rewards and income to go to one place.)

Call:

msg=
{
"act"="create_creator_account"
"devid"="",
"appid"="",
"devName"="",
"contactName"="",
"contactEmail"="",
"openseed"=""
}

Returns:

  • Success:

    {"creator_account":{"devID":"privateID","pubID":"publicID"}}

  • No OpenSeed User:

    {"creator_account":{"server":"no openseed user found"}}

  • Account exists:

    {"creator_account":{"devID":"exists","pubID":"exists"}}

#Create App or Game:

Creates Application and associates it with the creator account.

note

Name spacing is required but not yet automatic. So the developer will need to use the now standard reverse DNS style name space.

example: com.developer.applicaiton

Call:

msg=
{
"act"="create_app"
"devid"="",
"appid"="",
"appName"=""
}

Returns:

  • Success"

    {"appID":"privateID","pubID":"publicID"}

  • Exists:

    {"appID":"exists","pubID":"exists"}

#Create Profile:

Creates the long form profile for users,developers, and applications. Called after the ids have been created.

caution

This will change to match the datasets within OpenSeed.

Call:

msg=
{
"act":"set_profile",
"devid":"",
"appid":"",
"token":"",
"data1":"",
"data2":"",
"data3":"",
"data4":"",
"data5":"",
"type":""
}

Returns:

  • Sucess:

    {"profile":"updated"}

#History

info

Recording history is completely volentary, and will be important for developers to inform their users on what historical data they are sending to OpenSeed.

Standard Types:

OpenSeed has various built-in types for historical data which streamlines input and standardizes output. We currently represent these as numeric values but that may change in the future.

Type ValueCategoryRequired data
1Program Start"program_start":"program"
2Program Stop"program_stop":"program"
3Playing Music"playing":{"song":"title","artist":"artist"}
4Purchase"purchase":"item"
5Download"download":"item"
6Linked"linked":"username"
7High Score"highscore":"score"
tip

Non standard types can be manually specified by the developer.

#Add to History:

Adds to the history record for the user.

Call:

msg=
{
"act":"update_history",
"token":"",
"type":"",
"appPub":"",
"data":""
}

Returns:

  • Success

    {"history":"updated"}

#Get History:

Returns the history for a specific account. Optionally you may define a specific application(apprange) and a total count(count).

Call:

msg=
{
"act":"get_history",
"devid":"",
"appid":"",
"account":"",
"apprange":"",
"count":""
}

Returns:

{"history":[{"history":"type","item":"data"}]}

#Hive Account Functions

#Hive Login:

This creates an openseed user based on the hive profile retrieved on verification of their hive account, gives openseed posting rights, and gives the user the option to have their keys stored. This is the basis for logins and other hive/openseed functions. Call:

msg=
{
"act":"hive_login",
"devid":"",
"appid":"",
"account":"",
"postkey":"",
"store":boolean,
"import
}

Returns:

  • OpenSeed user exists

    {"interconnect":"connected","account_auth":"openseed","keystored":"true or false"}

  • New User

    {comming soon}

#Hive Connect:

Used to connect an existing openseed user to a hive profile retrieved on verification of the hive account, gives openseed posting rights, and gives the user the option to have their keys stored. This is the basis for logins and other hive/openseed functions. Call:

msg=
{
"act":"hive_connect",
"devid":"",
"appid":"",
"account":"",
"postkey":"",
"store":boolean,
}

Returns:

  • Success

    {"interconnect":"connected","account_auth":"openseed","keystored":"true or false"}

#Hive Delegation:

If the user has allowed OpenSeed to store their keys we can quickly add or remove posting rights through the hive_delegation function. If they don't have their keys saved by OpenSeed we will need to ask them for their posting key to finish the process. We derive the application we are allowing/disallowing via appid.

note

This function will attempt to streamline login prompts similar to steemconnect or hivesigner, but steps will need to be taken to create openseed accounts using hive_openseed_interconnect if the developer is hoping to get the login to a point where it behaves like other login managers. (i.e. facebook,google,etc.)

Call:

msg=
{
"act":"hive_delegation",
"devid":"",
"appid":"",
"account":"",
"postkey":"" -or- "passphrase":"",
"option":"allow"/"disallow"
}

Returns:

{delegation:{"account":"","rights":"posting","to":"hiveapp"}

#Flush Keys (Hive):

Not needed for most operations as the deletion of keys is a part of every request that requires one (unless stored is set true). However, when OpenSeed creates an account for a user the keys are stored for fast access. This function can be used to remove the keys from our server. Call:

msg=
{
"act":"hive_flush_keys",
"devid":"",
"appid":"",
"hiveaccount":"",
}

Returns:

{"removed":hive account}

#Send Payment (Hive):

"devid":"",
"appid":"",
"account":"",
"apprange":"",
"count":""
}

Returns:

{"payment":{"sent":"ammount","in":"tokentype","to:"account"}}
