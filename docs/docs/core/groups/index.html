Groups

Current existing functions dealing with OpenSeed Groups. Please read the About section to understand how to use the below in a fully formatted command. Not all these functions are finalized and there will be changes while we work toward our first release. However, these should represent most of the major features.

info

Groups in OpenSeed are treated similar to user accounts sporting their own unique ids and permissions. In this way the developer can use the group functions to achieve a far greater variety of use cases. From the simple chat groups (public and private) to guilds with shared items and resources. The sky is the limit as far as how you use the group but the first step is setting one up.

#General Group Functions

#Creating a Group

Creating a group will send a connection request on behalf of the group to those set by the "allowed" variable. These users must already be connected to the owner of the group to be included in the connection request creation.

Call

msg = {"act":"create_group",
"appPub":"",
"devPub":"",
"token":"",
"title":"",
"allowed":"comma,seperated,list",
"denied":"also,comma,seperated"
}

Returns

  • Success

    '{"group":{"token":"","title":"","allowed":"","denied":""}}'

  • Error (Exists)

    '{"group":{"title":"exists"}}'

#Deleting a Group

warning

Deleting a group is final, all records (chat,connections,profiles,and roles) will be expunged. However, any data or service that extends the group in app_priv_data or app_pub_data will be left untouched.

Call

msg = {"act":"delete_group",
"appPub":"",
"devPub":"",
"token":"",
"title":""
}

Returns

  • Success

    '{"deleted_group":{"title":"group title"}}'

  • Error (ownership required)

    '{"deleted_group":{"error":"not the owner"}}'

#Listing Groups

Lists groups you are in, invited to, or the owner of.

Call

msg = {"act":"list_groups",
"appPub":"",
"devPub":"",
"token":""
}

returns

  • Success

    {"group_list":[{"group":title,"status":owner,user,pending,"allowed":[],"denied":[]}]}

  • No Groups

    {"group_list":[]}

#Security

Security is always important when dealing with groups. OpenSeed adopts the standard owner,admin,mod,user approach to make things easier for those using it.

RoleFunction
ownerFull rights to the group and is the primary contact for it.
adminAble to function as the owner for all function except re-adding kicked users or deleting the group
modAble to mute users and delete / hide messages if needed
userCan access the group as a normal user. Chat, post, delete their own content, etc.
note

There can be only one owner but an admin account can do anything the owner can do except re-add a user once they have been kicked from the group and delete the group.

warning

These functions are still in the works and have not been tested or finalized.

#Misc

#Public Invites

Public invites are specially created URLs that will allow users to by-pass the need to be connected to any of the members of the group. Commonly used for larger groups or open forums. The users will still need to have an OpenSeed account to access the group, but will be added once the authentication has taken place.
