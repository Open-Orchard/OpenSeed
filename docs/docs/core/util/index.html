Utilities

Functions that are designed to help developers but don't fit in other libraries.

#Image Functions

As a part of OpenSeeds turnkey solutions we store and serve various media types. Similar to how the Music functions reencode the music to an opensource file type the Image functions convert known formats into their opensource equivalants. Jpegs are converted to PNGs, gifs are converted to webm. This ensures the maxium use case for the item as well as saving harddrive space in many instances.

#Storing an Image

This function is used to grab an image from any source and then import it into openseed. This is useful for preexisting published images and not to be intended for uploads. Image store takes the original file and creates three versions of the file. Low-res(1024x),Mid-res(2048x),High-res(4096x). These files will be stored on the server and can be accessed by others using the image_get function.

Call

msg= {"act":"image_store",
"devId":"",
"appId":"",
"url":""
}

Returns

#Getting Image

Use get image to return the hash of the image based on the quality (size) you are needing for your application. The url is considered the "source" of the image so knowing the original hash or url is important.

note

We will be adding other ways to reference the image in the future.

Call

msg= {"act":"image_get",
"devId":"",
"appId":"",
"url":"",
"size":"low/medium/high/original"
}

Returns
